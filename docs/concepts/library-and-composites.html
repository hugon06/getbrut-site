Library & composites

Reuse is first-class in Brut. A pipeline you've perfected shouldn't be rebuilt by hand on the next job — collapse it, save it, drop it in anywhere.

#Groups

Select two or more nodes and press Ctrl+G (or Group in the floating selection bar) to wrap them in a labeled frame. A group is purely visual — it moves as a unit and keeps things tidy. ✕ on the frame ungroups.

#Composites

A composite is a subgraph collapsed into a single node (select nodes → 📦 Collapse to composite). Cut wires become labeled ports; the inner nodes are still real (just hidden), so caching, pins and ledger history keep working inside the composite exactly as outside.

  • Open / Close — ⤢ opens a composite into a live frame you can edit inside, then ⇱ packs it back up (a perfect round-trip; ports regenerate from the cut wires).
  • Run — ▶ on a composite runs its whole inside plus everything upstream.
  • Explode — 💥 permanently dissolves it back into loose nodes.

Composites are how the four-agent cinema crew becomes one droppable "Cinema Team" node.

#The global library

Composites and whole-graph templates can be saved to a global library — deliberately cross-project, so the same shelf appears on every project.

  • Save a composite — 💾 Save to library on the composite node. Same name = a new version (everything is versioned; any past version is retrievable).
  • Save a graph as a template — File ▾ → Save graph as template….
  • Browse — File ▾ → Browse library…, or find saved composites right in the Tab add-node menu under Library. Dropping one in gives it fresh ids (separate caches, no collisions).

Library items carry an optional category (a shelf label you pick on save) and hold sanitized recipes only — prompts, models and wiring. No images, no caches, no keys are ever written into a library item, so they're safe to share.

Some items ship with Brut — like the Cinema Crew template (the whole cinema pipeline, pre-wired). Built-ins can't be deleted; saving your own item under the same name shadows the shipped one.

#Templates vs composites

  • A composite is a node — a reusable building block you drop into an existing graph.
  • A template is a whole graph — a starting point for a new job, opened fresh from the library.
