Install & first run

Brut runs on Windows as two local processes — the engine (Python) and the app (a built web UI the engine serves). The launcher handles both.

#Requirements

  • Windows 10/11.
  • Python 3.11+ on your PATH (the launcher bootstraps everything else).
  • A billing-enabled API key for at least one provider (see Providers). A free-tier key usually has zero image quota — a first-call 429 means billing isn't enabled, not rate-limiting.

#First launch

  1. Unzip the Brut seat pack anywhere on your disk.
  2. Double-click Brut.cmd. On the first run it creates a local Python environment, installs dependencies, builds the UI, starts the engine minimized, and opens http://127.0.0.1:8723/app/ in your browser.
  3. Subsequent launches just check the engine and open the browser — a second or two.

Second seat in the studio? All another machine needs is Python and the same folder — the launcher rebuilds its own environment. Your work travels separately, as a .brutpack file or a NAS-rooted project, never in the seat zip.

#Sign in

Brut requires a free account (it's how we count seats during the beta — nothing about your prompts, images or keys is ever sent). On first launch you'll hit a sign-in screen:

  • Continue with Google or GitHub, or
  • Magic link — enter your email and open the link it sends you in the same browser.

Sign-in is required, but it never bricks you: once you're in, a 30-day offline grace keeps you working through flights and outages, and if the sign-in server is ever unreachable a "continue for now" option appears. The engine holds your session; the browser keeps nothing.

#Add your provider keys

Click ⚙ Keys in the toolbar (or the setup wizard walks you through it on first run). For each provider, paste your key and Brut validates it on save. Keys are stored in your OS keychain — never in a graph, never sent anywhere but the provider.

You can also set keys via environment variables before launching the engine (GEMINI_API_KEY, OPENAI_API_KEY, etc.) — handy for a shared workstation.

See each provider's page for where to get a key and its billing quirks. The quickest start is Google Gemini — a gemini-2.5-flash-image render is about $0.04.

#The functionality tour

Right after the setup wizard, Brut walks you through the interface once — a short spotlight tour of the canvas, the node families, projects, running & caching, and where your keys live. Skip it any time with Esc, and relaunch it whenever you like from File ▾ → Functionality tour.

#Found a bug? Tell us from inside the app

The 💬 button in the toolbar opens Send feedback — a bug, an idea, anything. Reports go straight to the team, tied to your account email so we can reply. Diagnostics (app/engine version, OS, recent error messages) are opt-in, and the modal shows the exact payload before you send — never your graphs, images, prompts or keys. Details in Privacy.

#If the engine is offline

The app shows an "engine offline" banner with a retry. Usually it means the engine process isn't running — relaunch Brut.cmd. The launcher serves the last built UI, so if you're developing and changed the app, rebuild to see it.
